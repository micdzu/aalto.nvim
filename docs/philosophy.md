# Philosophy

Aalto starts with a question:

> What should color do in code?

---

## Color as a Constraint

Color is a limited resource.

Most themes treat it as abundant. Aalto treats it as scarce.

Every color must justify its existence.

---

## Semantic Reduction

Instead of many categories, Aalto uses four:

- definition → structure
- string → data
- constant → values
- comment → context

Everything else is neutral.

This is not simplification for its own sake.

It is reduction to increase signal.

---

## Meaning over Syntax

Most themes color syntax.

Aalto colors meaning.

It is not designed to show tokens. It is designed to reveal structure.

Over time:

- definitions become anchors
- strings become data blocks
- constants become signals
- comments fade into context

You stop parsing. You start perceiving.

---

## Perception over Decoration

Aalto avoids:

- decorative highlighting
- per-language styling
- visual noise

Instead, it builds a stable perceptual hierarchy.

Color should guide the eye, not demand attention.

---

## Consistency as a Feature

The same role always looks the same.

Across:

- languages
- plugins
- UI elements

This consistency reduces cognitive load over time.

---

## Tradeoffs

Aalto is not optimized for:

- learning syntax (use a tutorial theme)
- expressive or colorful visuals (use Catppuccin, TokyoNight)
- per-language customization

It is optimized for:

- long sessions
- structural thinking
- quiet clarity

---

## Who Aalto Is NOT For

- You want rainbow parentheses or colorful brackets
- You want language-specific colors (Python vs Go should look different)
- You want high-contrast neon or pastel aesthetics
- You want to impress viewers on Twitch/YouTube

Aalto is for **users**, not **viewers**.

---

## Strict Mode

Strict mode removes all styling.

No bold. No italic.

Only semantic color remains.

This is Aalto in its most reduced form.

Use it when:

- You find _any_ styling distracting
- You use a variable-width font where bold changes metrics
- You want to test if the semantic model actually works

---

## Final Thought

> The best colorscheme is the one you stop noticing.

Aalto is not about making code more visible.

It is about making everything else less distracting.
